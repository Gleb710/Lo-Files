# 🚀 Руководство по разработке (Workflow команды)

Привет! Если ты читаешь этот файл, значит, ты собираешься внести свой вклад в разработку. Чтобы мы не путались в коде, не затирали работу друг друга и понимали, кто чем занят, мы используем строгий workflow на базе **GitHub Projects** и **Git Flow**. 

Пожалуйста, следуй этим правилам. Они берегут нервы всей команды.

---
## 📌 1. Работа с доской задач (Канбан)

Вся наша активность прозрачна и отображается на вкладке **Projects** проекта. Доска состоит из 5 колонок:
  1. `📌Задачи` — бэклог всех идей, фич и багов. Сюда задачи попадают автоматически при создании Issue.
  2. `📋Готов к разработке` — приоритетные задачи на ближайший спринт (неделю). Сюда карточки переносит админ.
  3. `✍️В процессе` — то, что пишется или дизайнится прямо сейчас.
  4. `🔍Проверка кода` — код написан, создан Pull Request, ждем код-ревью и тестов.
  5. `✅Готово` — задача закрыта, код успешно слит с проектом.

### Как начать работу над задачей:
  1. Зайди на доску проектов и посмотри колонку `📋Готов к разработке`.
  2. Выбери задачу, назначенную на тебя, и **руками перетащи её** в колонку `✍️В процессе`. Так вся команда поймет, что ты забрал её в работу.

---
## 🌿 2. Правила работы с ветками (Git Flow)

Прямые коммиты и пуши (`git push origin main` или `dev`) на GitHub **заблокированы на уровне настроек репозитория**. Вся работа ведется только через отдельные ветки.

### Как называть ветки:
Создавай новую ветку от актуального состояния ветки **`dev`**:
  * Для новых функций: `feature/название-фичи` (например: `feature/local-crypto`)
  * Для исправления багов: `bugfix/описание-проблемы` (например: `bugfix/fix-padding`)

---
## 🔄 3. Отправка кода и Автоматизация (Pull Requests)

Когда ты закончил работу над фичей локально и закоммитил изменения, пришло время отправить её на проверку:
  1. Сделай пуш своей ветки на GitHub.
  2. Создай **Pull Request (PR) строго в ветку `dev`**.
  3. **КРИТИЧЕСКИ ВАЖНО:** Обязательно свяжи свой PR с исходной задачей. 
       * *Как это сделать:* В описании PR напиши волшебное слово и номер задачи, например: `Closes #12` (где 12 — номер задачи на доске). Также привязать задачу можно через меню **Development** в правой панели страницы PR.
       * *Что произойдет:* Как только ты привяжешь PR, автоматика сама заберет твою карточку из колонки `✍️В процессе` и перекинет её в **`🔍Проверка кода`**.

---
## 🔍 4. Код-ревью и Финиш

1. Твой PR должен проверить как минимум **1 участник команды**.
2. **Если найдены ошибки:** Тестировщик или коллега нажмет *Request changes*. Робот автоматически вернет твою карточку в колонку `✍️В процессе`. Исправь замечания и сделай новый пуш.
3. **Если всё отлично:** Коллега нажимает *Approve*. После этого делается **Merge pull request**.
4. Как только ветка сливается с `dev`, GitHub сам закрывает задачу, и карточка автоматически улетает в колонку **`✅Готово`**.

Спасибо за соблюдение правил! Меньше хаоса — быстрее релиз! 🚀
